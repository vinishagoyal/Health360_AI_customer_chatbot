1. Executive Summary

Health360 is a health supplement business that needs faster, more consistent customer support for product questions, order questions, shipping concerns, returns, and complaints. The proposed solution is an AI-powered customer support experience on the Health360 website, supported by an admin dashboard and product catalog.

The prototype was built using a vibe-coding workflow: defining the business need, rapidly translating it into product screens and data flows, iterating through user feedback, and shipping working features quickly. The AI layer uses OpenRouter's free model API through a lightweight SDK-style client wrapper, allowing the chatbot to respond to customer queries using Health360 product and business context.

2. Business Problem

Problem Statement: Health360's support team faces a rising volume of repetitive product, order, and complaint queries. Without a self-service support layer, agents spend too much time answering common questions, customers wait longer for help, and the business risks inconsistent or unsafe supplement communication.

Customer Support Challenges

  • Customers repeatedly ask about supplement benefits, ingredients, dosage, stock, and product suitability.
  • Order-related questions such as shipping, tracking, returns, and delivery timing create avoidable support load.
  • Complaints and urgent concerns are not always categorized or escalated consistently.
  • Manual replies can vary by agent, which weakens brand consistency.
  • Health supplement businesses must avoid unsupported medical claims, diagnosis, or treatment advice.
  • Leadership needs better visibility into product demand, support volume, and customer query patterns.

3. Solution Offered Through Health360

The solution is a website-based AI customer support and ecommerce support layer. It combines a Health360 storefront, AI chatbot, cart experience, and admin dashboard. The goal is to deflect routine support queries while keeping humans involved for complaints, low-confidence questions, and medically sensitive conversations.

AI Chatbot Widget A floating website chatbot that answers product, order, shipping, returns, and general support questions.
Full-Screen Chatbot A larger testing and demo mode for validating chatbot behavior and example prompts.
Health360 Product Catalog A searchable and filterable storefront with Health360-branded supplement product images.
Admin Dashboard Product, customer, and order views for operations, inventory awareness, and support context.

Core Capabilities

  • Answer frequently asked customer questions instantly.
  • Recommend relevant supplement categories based on customer wellness goals.
  • Provide product names, categories, price, SKU, stock, and descriptions from the Health360 catalog.
  • Guide customers on order tracking, shipping, and returns.
  • Escalate risky, complaint-heavy, or medically sensitive queries to human support.
  • Show admins product, customer, order, revenue, and low-stock information.
  • Keep the storefront visually branded using generated Health360 supplement imagery.

4. How I Built It Using Vibe Coding

Vibe coding was used as the practical build approach: instead of starting with a long engineering specification, the product was shaped iteratively from the business problem, then refined through working screens, code changes, and feedback loops.

Vibe-Coding Workflow

  1. Understand the business pain: support overload, repetitive queries, and compliance risk.
  2. Turn the pain into user flows: customer asks a question, bot responds, risky cases escalate, admin tracks data.
  3. Build the usable interface first: storefront, product cards, filters, cart, chatbot widget, and admin pages.
  4. Add real data structure: products, customers, orders, and order items through Supabase-compatible helpers.
  5. Integrate AI behavior: create a system prompt from Health360 product and business context.
  6. Iterate visually: replace generic icons with Health360-branded supplement product images.
  7. Test and refine: run lint/build checks, restart the dev server, and adjust based on observed behavior.

Why Vibe Coding Worked for This Project

  • It allowed quick movement from concept to working demo.
  • Business feedback could be converted into UI and logic changes immediately.
  • The chatbot, storefront, and admin dashboard could evolve together instead of being designed separately.
  • It made the product easier to present because the demo showed real flows, not only documentation.

5. OpenRouter Free API Integration

The chatbot uses an OpenRouter-compatible integration through the project's lib/openrouter.ts module. This module behaves like a small SDK wrapper: it accepts chat messages, formats the API request, sends it to OpenRouter, validates the response, and returns clean assistant text back to the chatbot UI.

Integration Flow

  1. The user types a message in the Health360 chatbot widget.
  2. The chatbot loads Health360 context using buildChatContext().
  3. The system prompt is generated using product catalog, inventory summary, and business policies.
  4. The current conversation is sent to the OpenRouter chat completions endpoint.
  5. The assistant response is displayed in the chatbot with Markdown formatting.
  6. If the API fails, the chatbot falls back to safe predefined support responses.

Important Production Note

In the prototype, the OpenRouter key is referenced through a public environment variable so the demo can run quickly in the browser. For production, the AI call should move to a server-side API route so the key is not exposed to users. This would also allow better logging, rate limiting, moderation, and cost control.

7. Functional Requirements Covered

  • Customer can browse Health360 supplement products.
  • Customer can search, filter, sort, and view product details.
  • Customer can add items to cart and view totals.
  • Customer can ask the AI assistant product and order questions.
  • Bot can answer using Health360 catalog and business context.
  • Bot can fall back safely if AI service is unavailable.
  • Admin can view products, customers, orders, revenue, and low-stock information.
  • Admin can create/edit/delete product and customer records through Supabase helpers.
  • Order status can be reviewed and updated from the admin order page.
  • Product tables can export CSV reports.

9. Value Delivered

  • Reduces repetitive support load by answering common product and order questions.
  • Improves first response time with instant chatbot access.
  • Improves brand consistency through a controlled Health360 assistant voice.
  • Improves product discovery through a searchable, branded storefront.
  • Gives admins operational visibility into product, customer, and order data.
  • Creates a strong base for future WhatsApp, email, CRM, and reporting integrations.

10. Recommended Next Steps

  1. Move OpenRouter API calls from the browser to a secure server-side API route.
  2. Tighten Supabase RLS policies before production use.
  3. Add conversation logging with category, confidence, escalation status, and timestamps.
  4. Create an approved FAQ and compliance-safe product claim library.
  5. Add human handoff workflow for complaints, medical questions, and refund issues.
  6. Add analytics for deflection rate, escalation rate, response time, and top query categories.
